## Day 2

### Comparison Operators

- `==` Equal to
- `!=` Not equal to
- `>` Greater than
- `<` Less than
- `>=` Greater than or equal to
- `<=` Less than or equal to

Comparison operators always return a Boolean value (`True` or `False`).

### Things I Learned

- `=` assigns a value.
- `==` compares values.
- `"10" == 10` returns `False` instead of an error.
- `"Python" != "python"` returns `True` because Python is case-sensitive.
- `True == 1` and `False == 0` both evaluate to `True` in Python.

# Day 3

## If Statements

The `if` statement allows a program to make decisions.

Syntax:

```python
if condition:
    # code to execute
```

### Things I Learned

- Every `if` statement requires a condition.
- The condition must evaluate to either `True` or `False`.
- A colon (`:`) is required after the condition.
- Python uses indentation to determine which code belongs to the `if` block.
- If the condition is `False`, Python skips the indented block.

### Common Mistakes

- Forgetting the colon.
- Forgetting indentation.
- Using `=` instead of `==`.

# Day 4

## else Statements

The `else` statement runs when the `if` condition evaluates to `False`.

### Syntax

```python
if condition:
    # code runs if True
else:
    # code runs if False
```

### Things I Learned

- `else` always works with an `if` statement.
- Only one branch (`if` or `else`) executes.
- Proper indentation is required.
- Giving users feedback for both outcomes improves the user experience.

### Common Mistakes

- Forgetting the colon (`:`) after `else`.
- Incorrect indentation.
- Expecting both `if` and `else` to run.
